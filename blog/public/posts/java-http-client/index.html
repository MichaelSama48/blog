Kỷ Nguyên Mới Của Giao Tiếp: Java HttpClient và Sự Tinh Tế Của Asynchronous

Đã qua rồi cái thời dùng HttpURLConnection cồng kềnh. Hãy cùng khám phá một vũ khí mới trong Java 11 giúp bạn 'trò chuyện' với các API khác một cách mượt mà và đầy nghệ thuật.
June 3, 2024 · 4 min

Lời tạm biệt với quá khứ#

Nếu bạn là một lập trình viên Java lâu năm, chắc hẳn bạn đã từng trải qua nỗi đau mang tên HttpURLConnection. Cái tên dài ngoằng, cách sử dụng thì rườm rà, và cảm giác như bạn đang quay lại thời kỳ đồ đá mỗi khi phải viết 20 dòng code chỉ để gửi một cái GET request đơn giản.

Bạn phải mở connection, set method, check response code, đọc input stream line-by-line… Ôi chao, quá nhiều thủ tục hành chính cho một việc đơn giản là “Lấy dữ liệu”. Nhiều người đã bỏ cuộc và tìm đến những thư viện bên ngoài như Apache HttpClient hay OkHttp. Chúng tốt, nhưng việc thêm một đống dependency vào project đôi khi lại làm phình to file JAR của bạn một cách không cần thiết.

Nhưng ánh sáng đã đến! Kể từ Java 11, Oracle đã mang đến một món quà tuyệt vời: java.net.http.HttpClient. Nó không chỉ là một sự nâng cấp, nó là một cuộc lột xác. Nó hiện đại, API được thiết kế theo kiểu Fluent (chuỗi phương thức), hỗ trợ HTTP/2, và quan trọng nhất: nó hiểu được ngôn ngữ của sự Bất đồng bộ (Asynchronous).

Sức mạnh của sự chờ đợi (Synchronous vs Asynchronous)#

Hãy tưởng tượng bạn gọi điện đặt Pizza.

  • Synchronous (Đồng bộ): Bạn gọi điện, và bạn cứ giữ máy như vậy, không dám làm gì khác, chờ cho đến khi đầu dây bên kia xác nhận đơn hàng xong. Bạn bị “tê liệt” (Blocked) hoàn toàn trong khoảng thời gian chờ đó. Nếu bên kia bận 5 phút, bạn mất 5 phút cuộc đời.
  • Asynchronous (Bất đồng bộ): Bạn nhắn tin đặt hàng rồi vứt điện thoại sang một bên. Bạn đi xem tivi, đi tắm, làm đủ thứ việc. Khi nào quán Pizza xác nhận, điện thoại sẽ “ting” một tiếng báo hiệu (Callback/Future). Bạn không lãng phí một giây nào cả. Đây mới là cách người thông minh làm việc.

Java HttpClient mới cho phép bạn làm điều thứ hai đó một cách cực kỳ thanh thoát.

Synchronous - Cách cổ điển#

// Tạo một khách hàng ảo
HttpClient client = HttpClient.newHttpClient();

// Soạn một bức thư
HttpRequest request = HttpRequest.newBuilder()
      .uri(URI.create("https://api.vutru.com/thong-diep"))
      .GET()
      .build();

// Gửi thư đi và... ĐỨNG CHỜ (Block)
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println("Vũ trụ trả lời: " + response.body());

Asynchronous - Phong cách hiện đại#

// Gửi tin nhắn đi và... quên nó đi!
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
      .thenApply(response -> {
          // Khi nào có thư, dòng code này mới chạy
          return response.body();
      })
      .thenAccept(noiDung -> System.out.println("À, vũ trụ vừa nhắn lại: " + noiDung))
      .join(); // (Chỉ dùng join khi bạn thực sự cần chờ kết quả cuối cùng để tắt app)

Bạn thấy vẻ đẹp của nó chứ? .sendAsync() trả về ngay lập tức một cái CompletableFuture. Nó giống như một lời hứa (Promise trong JS): “Tao sẽ đi lấy dữ liệu cho mày, mày cứ làm việc khác đi, khi nào có tao sẽ báo”. Code của bạn không bị dừng lại, CPU được giải phóng để xử lý những việc quan trọng hơn. Đây là chìa khóa để xây dựng những hệ thống chịu tải cao, nơi mà mỗi luồng (Thread) đều quý như vàng.

JSON: Ngôn ngữ chung của loài người (và máy móc)#

Trong thế giới API, JSON là vua. Nó ngắn gọn, dễ đọc, và mọi ngôn ngữ đều hiểu. Nhưng HttpClient của Java lại khá “thuần khiết”, nó chỉ trả về String. Để biến String đó thành Object Java (ví dụ class User hay Product), chúng ta thường cần một người phiên dịch tài ba.

Không ai khác, đó chính là Jackson hoặc Gson. Đừng cố gắng parse JSON bằng tay (String manipulation) hay dùng Regex. Đó là con đường dẫn đến đau khổ và những lỗi lầm tai hại. Hãy để các thư viện chuyên dụng làm việc đó. Sự kết hợp giữa HttpClient (để vận chuyển) và Jackson (để phiên dịch) tạo nên một cặp đôi hoàn hảo cho mọi ứng dụng Java Backend hiện đại.

Lời kết#

Java đang thay đổi, nhanh hơn và hiện đại hơn chúng ta nghĩ. Việc nắm vững HttpClient mới không chỉ giúp code của bạn gọn gàng hơn, mà còn thể hiện tư duy của một lập trình viên biết tối ưu hóa tài nguyên hệ thống thông qua cơ chế Asynchronous. Hãy để code của bạn “thở”, đừng bắt nó phải chờ đợi vô ích. Hãy để CPU được tự do bay nhảy và làm những việc có ích hơn là ngồi chờ một gói tin từ server bên kia bán cầu.
