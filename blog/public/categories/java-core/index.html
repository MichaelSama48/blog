Làm Chủ Exception Handling: Để Code Của Bạn Không Bao Giờ 'Chết'

1. Bản chất của Exception trong JVM Khi một dòng code gây lỗi (ví dụ chia cho 0), JVM thực hiện các bước sau: Tạo một Object tương ứng (ví dụ ArithmeticException). Dừng luồng thực thi hiện tại. Tìm kiếm “Stack Trace” ngược từ trên xuống dưới xem có hàm nào catch lỗi đó không. nếu tìm thấy -> Nhảy vào block catch. Nếu không thấy -> In Stack Trace ra console và KILL THREAD ĐÓ. 2. Checked vs Unchecked: Cuộc tranh cãi thế kỷ Checked Exception (Lỗi nghiệp vụ) Kế thừa từ Exception. Bắt buộc xử lý (try-catch hoặc throws). Ý nghĩa: “Lỗi này có thể xảy ra do môi trường (mất mạng, hết ổ cứng), bạn hãy chuẩn bị phương án B đi”. Ví dụ: Đi rút tiền mà ATM hết tiền -> Nên báo lỗi để người dùng biết mà sang cây khác (Checked). Unchecked Exception (Lỗi lập trình) Kế thừa từ RuntimeException. Không bắt buộc xử lý. Ý nghĩa: “Lỗi này do mày code ngu (null pointer, sai index), sửa code đi chứ đừng có catch”. Ví dụ: Đi rút tiền mà thẻ ATM làm bằng giấy bìa -> Lỗi logic, không thể phục hồi (Unchecked). 3. Giải Phẫu Java Collections: Big-O và Cơ Chế Bên Trong

1. Độ phức tạp thuật toán (Big O Notation) Trước khi chọn Collection, bạn phải hiểu giá của nó. O(1): Tuyệt vời. Thời gian không đổi dù dữ liệu tăng lên 1 tỷ dòng. O(log n): Rất tốt. Tăng cực chậm (Binary Search). O(n): Tạm ổn. Dữ liệu tăng gấp đôi, thời gian tăng gấp đôi. O(n^2): Thảm họa. Dùng 2 vòng for lồng nhau. 2. ArrayList vs LinkedList: Cuộc chiến không hồi kết ArrayList (Mảng động) Bên trong: Là một mảng Object[]. Cơ chế: Khi mảng đầy, nó tạo mảng mới to hơn (thường là 1.5 lần), rồi copy dữ liệu cũ sang. -> Thao tác resize này rất tốn kém. Lời khuyên: Nếu biết trước số lượng, hãy khởi tạo capacity: new ArrayList<>(10000). LinkedList (Danh sách liên kết) Bên trong: Các Node trỏ lẫn nhau (Prev <-> Node <-> Next). Điểm yếu: Tốn bộ nhớ hơn (lưu thêm 2 con trỏ). Cache locality kém (các node nằm rải rác trong Heap, CPU khó cache). Thực tế: 99% trường hợp bạn nên dùng ArrayList. LinkedList chỉ thắng khi bạn cần insert/delete ở đầu list liên tục. 3. HashMap: Kiệt tác kỹ thuật Đây là cấu trúc dữ liệu quan trọng nhất. ...

JIT Compiler (Just-In-Time): Đây là vũ khí bí mật của Java.
